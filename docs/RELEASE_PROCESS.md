# Безопасная публикация релиза

Этот документ относится только к публичной витрине. Сборка выполняется в приватном репозитории и не должна переносить сюда исходный код или историю Git.

## Разрешённый процесс

1. В приватном контуре собрать и протестировать production-архив.
2. Проверить состав архива: без PDB, конфигов PROD, логов, ключей, токенов, исходников, Reactor-карт и внутренних документов.
3. Сформировать SHA-256.
4. Обновить `CHANGELOG.md` и публичную документацию вручную.
5. Запустить `.safety/verify-public-repo.ps1 -Mode Full`.
6. Создать GitHub Release только после явного подтверждения владельца.
7. Загрузить архив и файл SHA-256 как Release assets — не добавлять их в Git.
8. Скачать опубликованный архив обратно, повторно сверить SHA-256 и выполнить smoke test.

Запрещены subtree/filter-repo/export из приватного монорепозитория: они могут перенести удалённую историю или неожиданные файлы.
