# Правила безопасности публичного репозитория

Это публичный репозиторий документации и готовых релизов StabiLink Desktop. Он физически отделён от приватного репозитория исходного кода.

## Обязательные правила для ИИ и автоматизации

- Исходный код приложения запрещён в текущем состоянии и во всей Git-истории этого репозитория.
- Нельзя копировать сюда каталоги Core, Desktop, Backend, инфраструктуры, резервных копий или приватную Git-историю.
- Запрещены исходники, проекты сборки, PDB, конфиги, журналы, базы, ключи, токены, архивы исходников, патчи и Git bundles.
- Запрещено использовать `git add .`, `git add -A`, `git push --mirror`, `git push --all` и `--no-verify`.
- Разрешены только точные пути из `.safety/allowed-files.txt`.
- Перед каждым commit и push необходимо запускать `.safety/verify-public-repo.ps1 -Mode Full`.
- Единственный разрешённый remote — `sany86russ/StabiLink-Desktop`.
- Production-архив разрешено загружать только как GitHub Release asset после явного одобрения владельца. Бинарники и архивы запрещено добавлять в Git.
- Нельзя создавать или менять remote, публиковать Release либо менять видимость репозитория без явного подтверждения владельца в текущей сессии.

Если нужного файла нет в allowlist, необходимо остановиться и запросить решение владельца. Автоматически расширять allowlist запрещено.
