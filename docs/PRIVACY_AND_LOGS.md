# Приватность и журналы StabiLink

[[Вернуться к README]](../README.md)

## Для чего нужны журналы

Локальные журналы помогают понять:

- прошёл ли запуск приложения;
- какой модуль включался или завершался;
- успешно ли применена стратегия;
- произошла ли ошибка обновления;
- на каком этапе остановилась диагностика.

## Автоматическая защита

Перед записью StabiLink маскирует чувствительные категории данных: данные авторизации, токены, ключи, закрытые служебные адреса, подписочные ссылки и локальные пользовательские пути. Та же очистка применяется при формировании диагностического архива.

## Что нельзя публиковать

- QR-коды авторизации;
- комбинации подтверждения устройства до окончания входа;
- токены и ключи;
- подписочные URL;
- содержимое персонального профиля;
- полные необработанные журналы;
- скриншоты с личными сведениями без предварительной проверки.

## Как безопасно сообщить об ошибке

1. Повторите проблему один раз.
2. Откройте Control Center → «Логи».
3. Используйте встроенный экспорт.
4. Просмотрите архив перед отправкой.
5. В публичный Issue добавляйте только короткий очищенный фрагмент.
6. Полный архив передавайте через личный канал поддержки.

Даже при автоматической маскировке пользователь должен самостоятельно проверить материалы перед публикацией.
